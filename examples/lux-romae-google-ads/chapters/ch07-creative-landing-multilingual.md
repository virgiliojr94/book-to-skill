# Chapter 7: Creative, Landing Pages & Multilingual Execution

## Core Idea
A multilingual site is only an asset if the **paid structure respects it**. Match ad language → landing-page language → booking-engine language deliberately. Don't trust Google to improvise. Above-the-fold must answer the booking question in seconds.

## Frameworks Introduced

### Tiered Multilingual Rollout (the biggest budget-saving choice)

| Tier | Languages | Recommendation |
|---|---|---|
| **Core launch** | English, Italian | Must have dedicated campaigns, ads, and landing pages |
| **Expansion** | Spanish, French, German | Add when search volume + conversion quality justify |
| **Site-only at launch** | Remaining translated languages (from the 13-language site) | Keep pages live and ready, but **do not fragment paid spend until demand is proven** |

Why: thirteen thin campaigns produce low data density, slower learning, and weaker ad testing. Fewer, properly localised campaigns produce **faster learning + better Quality Score dynamics**.

### Language-Matching Rule
> Google may serve an ad in a **different language** from the user's query if it believes the user understands the ad language.

**Implication**: helpful for scale, dangerous for conversion rate. Lux Romae must create **separate ad-language groups** and route each to the **correct language subdirectory** — don't let Google improvise.

### Minimum Search Asset Stack
Ad assets contribute to Ad Rank. Use as many relevant asset types as possible.

| Asset type | Lux Romae use |
|---|---|
| **Sitelinks** | Book Direct, The Apartment, About Us, Explore Rome, Contact |
| **Callouts** | Book Direct, Flexible Options, Near Piramide Metro, Family Friendly |
| **Structured snippets** | Amenities: WiFi, Kitchen, Smart TV, Air Conditioning |
| **Call asset** | Show only during answered hours (use scheduling) |
| **Location asset** | Reinforce trust + mapping relevance if a Business Profile is active |
| **Price asset** | Only if rate examples + category labels can be kept accurate |
| **Promotion asset** | Only for real event-linked or seasonal offers |

**RSA quality rule**: provide multiple headlines and descriptions so Google can test combinations — minimum 8–10 headlines, 3–4 descriptions per ad group.

### High-Conversion Landing Page Structure (in this order)

| Zone | What to show |
|---|---|
| **Hero** | Apartment name, key location cue, occupancy cue, direct-booking CTA |
| **Trust strip** | Review rating, "book direct" reassurance, flexible policy summary |
| **Availability block** | Date picker or immediate "check dates" CTA |
| **Differentiators** | 2 bedrooms, family/couple suitability, transport access, neighbourhood advantages |
| **Policy block** | Flexible, Best Price, Reserve, Family Flexible explained simply |
| **Social proof** | Short review excerpts, photos, host credibility |
| **FAQ** | Check-in, cancellation, transport, payment, family suitability |

**First-screen test** — must answer in <3 seconds: *where is it, how many guests fit, why book direct, can I check dates now, what's the cancellation policy.* Storytelling goes below.

### Google Vacation Rentals Page/Feed Requirements (if pursued)
- **Listings with <5 images are blocked**; 8+ recommended
- **Structured data** required
- **Direct website link** required
- **Accurate prices + availability** essential
- Small inventory can use a **crawled listing feed** (no full XML feed needed)

### Other-Language Guidance (ES / FR / DE)
- **Localise for search intent, not just literal translation**
- **Stable headline structure across languages**:
  1. Property name OR direct-booking phrase
  2. Rome / location cue
  3. Differentiator
  4. Policy cue
- **Cancellation + payment wording must be human-reviewed** — machine translation here creates both legal risk and conversion-rate damage
- Language-matched ads **always land on the same language page**

## Key Concepts

- **RSA (Responsive Search Ad)**: ad format where Google mixes headlines/descriptions per query
- **Ad asset** (formerly extension): sitelink, callout, structured snippet, call, location, price, promotion — contribute to Ad Rank
- **Quality Score**: Google's keyword-level relevance score — improves with consistent ad/landing/keyword theme + CTR + LP experience
- **Language subdirectory**: per-language section of the site (`/en/`, `/it/`, `/es/`...) — required destination per language campaign
- **Crawled listing feed**: Google fetches small-inventory listings without a full XML feed; eligible for Vacation Rentals

## Mental Models

- **"Localise; don't translate."** Search intent in Italian is not just a translation of English search intent — sometimes the phrasing, the policy concern, even the implied trip occasion shifts.
- **"First screen = the booking interview."** If a visitor can't answer the 5 booking questions above the fold, the page is too slow.
- **"Assets are not optional decoration."** They're Ad Rank inputs and trust signals; missing assets cost cheap impression share.

## Anti-patterns

- **Running 13 thin language campaigns** because the site has 13 languages → data fragmentation kills learning
- **Sending non-English ads to an English landing page** because "they understand English" → measured drop in conversion rate
- **Machine-translating cancellation language** → legal + conversion damage at the worst place in the funnel
- **Long storytelling above the fold** → user can't see the booking-relevant info before bouncing
- **Submitting <5 images to Vacation Rentals** → listing blocked
- **Price assets with stale rates** → policy violations + trust damage

## Key Takeaways

1. **English + Italian first**, ES/FR/DE only when justified, rest at site-level only.
2. **Separate ad-language groups → matching landing pages.** Always.
3. **Full asset stack** (sitelinks, callouts, snippets, call, location; price/promotion only if accurate).
4. **8+ images on Vacation Rentals listings** if you pursue that channel; <5 is blocked.
5. **First-screen test** — answer the 5 booking questions in <3 seconds.
6. **Human-review cancellation + payment wording** in every non-core language.
7. **Stable headline ladder across languages**: property → location → differentiator → policy.
